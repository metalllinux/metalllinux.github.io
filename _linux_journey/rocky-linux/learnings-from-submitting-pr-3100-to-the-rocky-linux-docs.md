# Learnings from Submitting PR 3100 to the Rocky Linux Docs 

* If you have any supporting documentation that needs updating, create separate PRs for those. For example, a new piece of documentation is in its own PR and has been merged. Then any supporting documentation that needs to be updated and linked to that new piece of documentation, should be in a separate PR.  

* Don't lump multiple changes from different documents into one PR. Make sure these are separate PRs.
