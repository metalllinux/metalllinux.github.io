# Kompose (Kubernetes + Compose)

<a id="user-content-kompose-kubernetes--compose"></a>[](#kompose-kubernetes--compose)

[![Build Status Widget](../_resources/badge_73ba0c80e253495d9c89bd52af570e75.svg)](https://github.com/kubernetes/kompose/actions?query=workflow%3A%22Kompose+CI%22) [![Coverage Status Widget](../_resources/68747470733a2f2f636f766572616c6c_2bcde3be23244c218.svg)](https://coveralls.io/github/kubernetes/kompose?branch=main)[![GoDoc Widget](../_resources/68747470733a2f2f676f646f632e6f72_8ac7277836734ed0a.svg)](https://godoc.org/github.com/kubernetes/kompose)[![GoReportCard Widget](../_resources/68747470733a2f2f676f7265706f7274_670408a39ce94a119.svg)](https://goreportcard.com/report/github.com/kubernetes/kompose)

[![logo](../_resources/logo_90296af701fe424dadbdedcad7ed8b0a.png)](https://github.com/kubernetes/kompose/blob/main/docs/assets/images/logo.png)

`kompose` is a tool to help users who are familiar with `docker-compose` move to [Kubernetes](http://kubernetes.io). `kompose` takes a [Compose Specification](https://compose-spec.io/) file and translates it into Kubernetes resources.

`kompose` is a convenience tool to go from local Compose environment to managing your application with Kubernetes. Transformation of the [Compose Specification](https://compose-spec.io/) format to Kubernetes resources manifest may not be exact, but it helps tremendously when first deploying an application on Kubernetes.

## Use Case

<a id="user-content-use-case"></a>[](#use-case)

Convert [`compose.yaml`](https://raw.githubusercontent.com/kubernetes/kompose/main/examples/compose.yaml) into Kubernetes deployments and services with one simple command:

```shell
$ kompose convert -f compose.yaml
INFO Kubernetes file "frontend-service.yaml" created
INFO Kubernetes file "redis-leader-service.yaml" created
INFO Kubernetes file "redis-replica-service.yaml" created
INFO Kubernetes file "frontend-deployment.yaml" created
INFO Kubernetes file "redis-leader-deployment.yaml" created
INFO Kubernetes file "redis-replica-deployment.yaml" created
```

Other examples are provided in the *examples* [directory](https://github.com/kubernetes/kompose/blob/main/examples).

#### Binary installation

<a id="user-content-binary-installation"></a>[](#binary-installation)

Kompose is released via GitHub on a three-week cycle, you can see all current releases on the [GitHub release page](https://github.com/kubernetes/kompose/releases).

**Linux and macOS:**

```shell
# Linux
curl -L https://github.com/kubernetes/kompose/releases/download/v1.34.0/kompose-linux-amd64 -o kompose

# macOS
curl -L https://github.com/kubernetes/kompose/releases/download/v1.34.0/kompose-darwin-amd64 -o kompose

chmod +x kompose
sudo mv ./kompose /usr/local/bin/kompose
```

**Windows:**

Download from [GitHub](https://github.com/kubernetes/kompose/releases/download/v1.34.0/kompose-windows-amd64.exe) and add the binary to your PATH.

## Shell autocompletion

<a id="user-content-shell-autocompletion"></a>[](#shell-autocompletion)

We support Bash, Zsh and Fish autocompletion.

```shell
# Bash (add to .bashrc for persistence)
source <(kompose completion bash)

# Zsh (add to .zshrc for persistence)
source <(kompose completion zsh)

# Fish autocompletion
kompose completion fish | source
```

## Development and building of Kompose

<a id="user-content-development-and-building-of-kompose"></a>[](#development-and-building-of-kompose)

### Building with `go`

<a id="user-content-building-with-go"></a>[](#building-with-go)

**Requisites:**

1.  [make](https://www.gnu.org/software/make/)
2.  [Golang](https://golang.org/) v1.6 or later
3.  Set `GOPATH` correctly or click [SettingGOPATH](https://github.com/golang/go/wiki/SettingGOPATH) for details

**Steps:**

1.  Clone repository

```shell
$ git clone https://github.com/kubernetes/kompose.git $GOPATH/src/github.com/kubernetes/kompose
```

2.  Change directory to the cloned repo.

```shell
cd $GOPATH/src/github.com/kubernetes/kompose
```

3.  Build with `make`

```shell
$ make bin
```

4.  Or build with `go`

```shell
$ go build -o kompose main.go
```

5.  Test your changes

```shell
$ make test
```
